Chapter 1 – Intro to Digital Marketing & Web Fundamentals

This chapter (and its sub-sections 1.1 and 1.2) sets the foundation for the whole course: what digital marketing is about, how the web works under the hood, and the simple 3-part framework that every website must master.

1.0 – Big Picture of the Course

Key Ideas

  • The course gives a ceiling-level understanding of digital marketing – you learn how channels work and what decisions to make, not just which buttons to click.
  • Digital marketing supports overall business goals, not just “more clicks” or “more likes”.
  • Before using any channel, you need to understand:
    • How websites and the Internet actually work (Chapter 1.1)
    • What makes a website successful (traffic, conversion, revenue per conversion – Chapter 1.2)

1.1 – Technical Foundations of the Internet

URLs, Domains, and IP Addresses

  • When you type amazon.com into the browser, it creates a URL like https://www.amazon.com/.
  • The domain name is human-friendly. Behind the scenes, it is translated into an IP address so your request goes to the correct server.
  • The browser sends an HTTP(S) request to that IP address and receives a response (HTML, CSS, JS, etc.).

HTTP vs HTTPS

  • HTTP – HyperText Transfer Protocol; rules for how browsers and servers talk.
  • HTTPS – HTTP + encryption (TLS/SSL). Protects data from eavesdropping; now a ranking signal in Google.

Servers and Hosting

  • A web server is a computer that stores website files and serves them on request.
  • You can technically host a website yourself, but most businesses use hosting providers (e.g., shared hosting, VPS, cloud).

Cookies

  • Small pieces of data stored in the browser by the website.
  • Allow the server to recognize you on return visits (e.g., “keep me logged in”, cart contents, preferences).
  • Without cookies, every page view would look like a brand-new visitor.

Static vs Dynamic Content

  • HTML is the core markup language that defines page structure.
  • Static pages – same content for every visitor (pure HTML files).
  • Dynamic pages – server-side languages (Java, PHP, Python, etc.) generate HTML on the fly, possibly using cookies and user data.
  • Dynamic behavior is what allows:
    • Personalized recommendations
    • Shopping carts
    • “Welcome back, Peggy” style greetings

1.2 – The Framework of Website Success

Every successful website excels at three things:

Traffic Conversion Revenue per Conversion

1) Traffic – Getting People to the Site

  • Direct – people type your URL, use a bookmark, or use your app.
  • Referral – clicks from other websites (news, blogs, partner sites).
  • Search – clicks from search engines:
    • Paid search – ads in search results (Google Ads).
    • Organic search – unpaid results driven by SEO.
  • Email – broadcast emails that bring people back.

2) Conversion – Turning Visitors into Something Valuable

  • Retail website – conversion = purchase.
  • Service business site – conversion = inquiry/lead (form submission, phone call).
  • Media site – conversion = page views, time on site, ad impressions.
  • Social media site – conversion = engagement, content creation.
  • Affiliate site – conversion = click-through to partner websites.

Important: Analytics only measures the measurable conversion (e.g., order placed), not necessarily the true “moment of decision” which may have happened earlier.

3) Revenue per Conversion – Making Each Conversion Worth More

  • Upsells – suggest complementary products during checkout.
  • Higher-priced alternatives – encourage customers to choose premium options.
  • Affiliate & ad optimization – show the most profitable ads/partners to the right user segments.

Final Cheat Sheet

  • To improve a site, always ask:
    • Do I need more traffic?
    • Do I need a higher conversion rate?
    • Can I increase revenue per conversion?
  • Under everything: understand how URLs, servers, cookies, and HTTP/HTTPS work.
